# User Manual — ระบบแจ้งซ่อม (Maintenance Request System)

**Version:** 2.0 | **Date:** April 2026

---

## 1. การเข้าสู่ระบบ

1. เปิดเว็บไซต์ระบบแจ้งซ่อม
2. กรอก ชื่อผู้ใช้ และ รหัสผ่าน
3. กดปุ่ม "เข้าสู่ระบบ"
4. หากข้อมูลถูกต้อง จะเข้าสู่หน้า Dashboard

**บัญชีทดสอบ:**

**หมายเหตุ:** Session หมดอายุใน 2 ชั่วโมง หากไม่มีการใช้งานจะ auto logout

## 2. Dashboard

หน้าแรกหลัง login แสดงภาพรวมทั้งหมด:

- **Stat Cards:** จำนวน Case ทั้งหมด, รอดำเนินการ, กำลังดำเนินการ, เสร็จสิ้น, เกิน SLA
- **กราฟสถานะ:** Doughnut chart แสดงสัดส่วนสถานะ — กดที่ส่วนสีเพื่อดูรายการ
- **กราฟประเภท:** Bar chart แสดงจำนวนตามประเภท — กดที่แท่งเพื่อดูรายการ
- **ภาระงาน:** กราฟแสดงจำนวนงานต่อคน — กดเพื่อดูรายละเอียด
- **SLA Overdue:** ตารางรายการที่เกิน SLA — กดเพื่อเปิดรายละเอียด
- **ค้นหาขั้นสูง:** กดปุ่ม ⚙️ เพื่อ filter ตามสถานะ/ประเภท

## 3. การสร้าง Case (แจ้งซ่อม)

1. ไปที่เมนู "แจ้งซ่อม"
2. กดปุ่ม "+ แจ้งซ่อมใหม่" (เปิด tab ใหม่)
3. กรอกข้อมูล:
   - **หัวข้อ Case** (จำเป็น) — เช่น "แอร์ไม่เย็น"
   - **รายละเอียด** — อธิบายปัญหา
   - **หมวดหมู่ Lv.1** (จำเป็น) — เลือกประเภทหลัก
   - **หมวดหมู่ Lv.2** — เลือกประเภทย่อย (แสดงหลังเลือก Lv.1)
   - **สถานที่** (จำเป็น) — เลือกจาก dropdown (Lv.1 > Lv.2)
   - **ลำดับความสำคัญ** — ต่ำ / ปานกลาง / สูง / เร่งด่วน
   - **ผู้แจ้ง** — ชื่อ + เบอร์ติดต่อ
   - **แนบไฟล์** — รูปภาพ, วีดีโอ, PDF, Word, Excel
4. กดปุ่ม "สร้าง Case"
5. SLA จะคำนวณอัตโนมัติจากประเภทย่อยที่เลือก

**หมายเหตุ:** ช่องที่มี * สีแดง คือช่องที่จำเป็นต้องกรอก หากไม่กรอกจะแสดงข้อความแจ้งเตือน

## 4. การดูรายละเอียด Case

1. ไปที่เมนู "แจ้งซ่อม"
2. กดที่แถวของ Case ที่ต้องการ (เปิด tab ใหม่)
3. หน้ารายละเอียดแสดง:
   - ข้อมูล Case ครบทุก field
   - ปุ่ม Workflow ตามสถานะและสิทธิ์
   - ผลการแก้ไข (สาเหตุ + วิธีแก้ไข)
   - ไฟล์แนบ
   - ประวัติการดำเนินการ (Timeline)

## 5. Workflow การจัดการ Case

### สำหรับ Manager:
| สถานะ Case | ปุ่มที่ใช้ได้ | ผลลัพธ์ |
|-----------|------------|--------|
| PENDING | มอบหมาย | เลือก member → ASSIGNED |
| PENDING | รับเอง | assign ให้ตัวเอง → ASSIGNED |
| PENDING | ยกเลิก | กรอกเหตุผล → CANCELLED |
| RESOLVED | Approve | → APPROVED |
| RESOLVED | Reject | กรอกเหตุผล → REJECTED |
| APPROVED | Complete | → COMPLETED |

### สำหรับ Member:
| สถานะ Case | ปุ่มที่ใช้ได้ | ผลลัพธ์ |
|-----------|------------|--------|
| ASSIGNED | ส่งผลแก้ไข | กรอกสาเหตุ + วิธีแก้ → RESOLVED |
| REJECTED | ส่งผลแก้ไข | แก้ไขใหม่ → RESOLVED |

## 6. การค้นหา Case

- **ค้นหาด่วน:** พิมพ์รหัส, หัวข้อ, หรือสถานที่
- **ค้นหาขั้นสูง:** กดปุ่ม ⚙️ แล้ว filter ตาม:
  - สถานะ, ประเภท, ผู้รับผิดชอบ
  - ช่วงวันที่
  - เฉพาะ SLA Overdue

## 7. รายงาน

1. ไปที่เมนู "รายงาน"
2. เลือก ปี/เดือน ที่ต้องการ
3. ดูกราฟสถานะ + ประเภท + ตารางสรุป
4. Export:
   - กดปุ่ม "Export PDF" → ดาวน์โหลดไฟล์ PDF
   - กดปุ่ม "Export Excel" → ดาวน์โหลดไฟล์ Excel

## 8. นำเข้า Case จาก Excel

1. ไปที่เมนู "แจ้งซ่อม" → กดปุ่ม "นำเข้า Case"
2. ดาวน์โหลด Template Excel
3. กรอกข้อมูลตาม Template
4. อัพโหลดไฟล์ → ตรวจสอบ Preview
5. กดปุ่ม "นำเข้า"

## 9. การตั้งค่า (Settings)

- **การแจ้งเตือน:** เปิด/ปิด แจ้งเตือนอีเมล, แจ้งเตือนในระบบ
- **ภาษา:** เลือก ไทย หรือ English
- **จำนวนรายการต่อหน้า:** 10 / 25 / 50

## 10. การจัดการระบบ (Admin เท่านั้น)

### จัดการผู้ใช้
- เพิ่ม/แก้ไข/ลบ ผู้ใช้
- กำหนด Role, email, เบอร์โทร, แผนก, ตำแหน่ง

### จัดการ Role
- สร้าง/แก้ไข Role
- กำหนด Permission ผ่าน Tree Node (เลือกทั้งกลุ่มหรือรายตัว)

### ประเภท Case
- จัดการประเภทหลัก (Lv.1) + ประเภทย่อย (Lv.2)
- กำหนด SLA (วัน) ต่อประเภทย่อย

### Master Data
- จัดการสถานที่ (Lv.1: ภายใน/ภายนอก → Lv.2: ชั้น/ห้อง)

### Interface Log
- ดูประวัติ API call ทั้งหมด
- กดดูรายละเอียด Request/Response Body (JSON)

### Login Log
- ดูประวัติการเข้าสู่ระบบ (สำเร็จ/ไม่สำเร็จ)
- แสดง IP, User Agent, เหตุผลที่ fail
