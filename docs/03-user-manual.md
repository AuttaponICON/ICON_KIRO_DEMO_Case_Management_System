# คู่มือการใช้งาน — ระบบแจ้งซ่อม

เวอร์ชัน 2.0 | วันที่ เมษายน 2569

## 1. การเข้าสู่ระบบ

1. เปิดเว็บไซต์ระบบแจ้งซ่อม
2. กรอก ชื่อผู้ใช้ และ รหัสผ่าน
3. กดปุ่ม "เข้าสู่ระบบ"
4. หากข้อมูลถูกต้อง จะเข้าสู่หน้าแดชบอร์ด

บัญชีทดสอบ:
| ชื่อผู้ใช้ | รหัสผ่าน | บทบาท |
|----------|----------|------|
| admin | 1234 | ผู้ดูแลระบบ |
| manager1 | 1234 | ผู้จัดการ |
| member1 | 1234 | สมาชิก |
| member2 | 1234 | สมาชิก |
| vip1 | 1234 | VIP |

หมายเหตุ: เซสชันหมดอายุใน 2 ชั่วโมง หากไม่มีการใช้งานจะออกจากระบบอัตโนมัติ

## 2. แดชบอร์ด

หน้าแรกหลังเข้าสู่ระบบ แสดงภาพรวมทั้งหมด:

- การ์ดสรุป: จำนวน Case ทั้งหมด, รอดำเนินการ, กำลังดำเนินการ, เสร็จสิ้น, เกิน SLA
- กราฟวงกลมสถานะ: แสดงสัดส่วนสถานะ กดที่ส่วนสีเพื่อดูรายการ
- กราฟแท่งประเภท: แสดงจำนวนตามประเภท กดที่แท่งเพื่อดูรายการ
- กราฟภาระงาน: แสดงจำนวนงานต่อคน กดเพื่อดูรายละเอียด
- ตาราง SLA เกินกำหนด: รายการที่เกิน SLA กดเพื่อเปิดรายละเอียด
- ค้นหาขั้นสูง: กดปุ่มเฟืองเพื่อกรองตามสถานะ/ประเภท

เมื่อกดที่กราฟ จะแสดงรายการ Case ในส่วนนั้น พร้อมจำนวน
เมื่อกดที่ Case ในรายการ จะเปิดแท็บใหม่ไปหน้ารายละเอียด

## 3. การสร้าง Case (แจ้งซ่อม)

1. ไปที่เมนู "แจ้งซ่อม"
2. กดปุ่ม "+ แจ้งซ่อมใหม่" (เปิดแท็บใหม่)
3. กรอกข้อมูลตามส่วนต่างๆ:

ส่วนที่ 1 - ข้อมูล Case:
- หัวข้อ Case (จำเป็น) เช่น "แอร์ไม่เย็น"
- รายละเอียดของ Case อธิบายปัญหาที่พบ

ส่วนที่ 2 - หมวดหมู่:
- เลือกหมวดหมู่หลัก (จำเป็น) เช่น ไฟฟ้า, ประปา, แอร์
- เมื่อเลือกแล้ว จะแสดงหมวดหมู่ย่อยให้เลือก พร้อมแสดง SLA แต่ละรายการ

ส่วนที่ 3 - สถานที่และลำดับความสำคัญ:
- เลือกสถานที่จาก dropdown (จำเป็น) แบ่งเป็น ภายใน/ภายนอก
- เลือกลำดับความสำคัญ: ต่ำ / ปานกลาง / สูง / เร่งด่วน

ส่วนที่ 4 - ข้อมูลผู้แจ้ง:
- ชื่อผู้แจ้ง (กรอกอัตโนมัติจากผู้ใช้ที่เข้าสู่ระบบ)
- เบอร์ติดต่อ

ส่วนที่ 5 - แนบไฟล์:
- กดเพื่อเลือกไฟล์ รองรับ รูปภาพ, วีดีโอ, PDF, Word, Excel
- แสดงตัวอย่างรูป + ไอคอนตามประเภทไฟล์ + ขนาดไฟล์
- กดปุ่ม X เพื่อลบไฟล์ที่ไม่ต้องการ

4. กดปุ่ม "สร้าง Case"
5. SLA จะคำนวณอัตโนมัติจากหมวดหมู่ย่อยที่เลือก

หมายเหตุ: ช่องที่มีเครื่องหมาย * สีแดง คือช่องที่จำเป็นต้องกรอก หากไม่กรอกจะแสดงข้อความแจ้งเตือนสีแดง

## 4. การดูรายละเอียด Case

1. ไปที่เมนู "แจ้งซ่อม"
2. กดที่แถวของ Case ที่ต้องการ (เปิดแท็บใหม่)
3. หน้ารายละเอียดแสดง:
   - ข้อมูล Case ครบทุกส่วน (เหมือนหน้าสร้าง)
   - ปุ่มดำเนินการตามสถานะและสิทธิ์ของผู้ใช้
   - ผลการแก้ไข (สาเหตุ + วิธีแก้ไข)
   - ไฟล์แนบ
   - ประวัติการดำเนินการ (แสดงเป็นเส้นเวลา)

## 5. ขั้นตอนการทำงาน Case

### สำหรับผู้จัดการ:
| สถานะ Case | ปุ่มที่ใช้ได้ | ผลลัพธ์ |
|-----------|------------|--------|
| รอดำเนินการ | มอบหมาย | เลือกสมาชิก สถานะเปลี่ยนเป็น มอบหมายแล้ว |
| รอดำเนินการ | รับเอง | มอบหมายให้ตัวเอง |
| รอดำเนินการ | ยกเลิก | กรอกเหตุผล สถานะเปลี่ยนเป็น ยกเลิก |
| ส่งผลแก้ไขแล้ว | อนุมัติ | สถานะเปลี่ยนเป็น อนุมัติแล้ว |
| ส่งผลแก้ไขแล้ว | ปฏิเสธ | กรอกเหตุผล สถานะเปลี่ยนเป็น ถูกปฏิเสธ |
| อนุมัติแล้ว | เสร็จสิ้น | สถานะเปลี่ยนเป็น เสร็จสิ้น |

### สำหรับสมาชิก:
| สถานะ Case | ปุ่มที่ใช้ได้ | ผลลัพธ์ |
|-----------|------------|--------|
| มอบหมายแล้ว | ส่งผลแก้ไข | กรอกสาเหตุ + วิธีแก้ สถานะเปลี่ยนเป็น ส่งผลแก้ไขแล้ว |
| ถูกปฏิเสธ | ส่งผลแก้ไข | แก้ไขใหม่ สถานะกลับเป็น ส่งผลแก้ไขแล้ว |

## 6. การค้นหา Case

- ค้นหาด่วน: พิมพ์รหัส, หัวข้อ, หรือสถานที่ในช่องค้นหา
- ค้นหาขั้นสูง: กดปุ่มเฟืองแล้วกรองตาม:
  - สถานะ
  - ประเภท
  - ผู้รับผิดชอบ
  - ช่วงวันที่
  - เฉพาะ SLA เกินกำหนด

## 7. รายงาน

1. ไปที่เมนู "รายงาน"
2. เลือก ปี/เดือน ที่ต้องการ
3. ดูกราฟสถานะ + ประเภท + ตารางสรุป
4. ส่งออก:
   - กดปุ่ม "Export PDF" ดาวน์โหลดไฟล์ PDF
   - กดปุ่ม "Export Excel" ดาวน์โหลดไฟล์ Excel

## 8. นำเข้า Case จาก Excel

1. ไปที่เมนู "แจ้งซ่อม" กดปุ่ม "นำเข้า Case"
2. ดาวน์โหลด Template Excel
3. กรอกข้อมูลตาม Template
4. อัพโหลดไฟล์ ตรวจสอบตัวอย่างข้อมูล
5. กดปุ่ม "นำเข้า"

## 9. การตั้งค่า

- การแจ้งเตือน: เปิด/ปิด แจ้งเตือนอีเมล, แจ้งเตือนในระบบ
- ภาษา: เลือก ไทย หรือ English
- จำนวนรายการต่อหน้า: 10 / 25 / 50

## 10. การจัดการระบบ (เฉพาะผู้ดูแลระบบ)

### จัดการผู้ใช้
- เพิ่ม/แก้ไข/ลบ ผู้ใช้
- กำหนด บทบาท, อีเมล, เบอร์โทร, แผนก, ตำแหน่ง

### จัดการบทบาท
- สร้าง/แก้ไข บทบาท
- กำหนดสิทธิ์ผ่าน Permission Tree (เลือกทั้งกลุ่มหรือรายตัว)

### ประเภท Case
- จัดการหมวดหมู่หลัก (ระดับ 1) + หมวดหมู่ย่อย (ระดับ 2)
- กำหนด SLA (วัน) ต่อหมวดหมู่ย่อย

### ข้อมูลหลัก (Master Data)
- จัดการสถานที่ (ระดับ 1: ภายใน/ภายนอก ระดับ 2: ชั้น/ห้อง)

### บันทึก API (Interface Log)
- ดูประวัติการเรียก API ทั้งหมด
- กดดูรายละเอียดข้อมูลที่ส่ง/ตอบกลับ (JSON)

### บันทึกการเข้าสู่ระบบ (Login Log)
- ดูประวัติการเข้าสู่ระบบ (สำเร็จ/ไม่สำเร็จ)
- แสดง IP, User Agent, เหตุผลที่ล้มเหลว
