# Requirements: ระบบแจ้งซ่อม (Maintenance Request System)

## Requirement 1: Authentication (การเข้าสู่ระบบ)
### User Stories
- ในฐานะผู้ใช้งาน ฉันต้องการเข้าสู่ระบบด้วยชื่อผู้ใช้และรหัสผ่าน เพื่อเข้าถึงระบบแจ้งซ่อม
- ในฐานะผู้ใช้งาน ฉันต้องการออกจากระบบได้ เพื่อความปลอดภัย

### Acceptance Criteria
1. แสดงหน้า Login พร้อมฟอร์มกรอกชื่อผู้ใช้และรหัสผ่าน
2. ตรวจสอบข้อมูลเข้าสู่ระบบ หากถูกต้องนำไปยังหน้า Dashboard
3. หากข้อมูลไม่ถูกต้อง แสดงข้อความแจ้งเตือน
4. หากยังไม่ได้ login ไม่สามารถเข้าถึงหน้า Dashboard ได้ (redirect กลับ Login)
5. มีปุ่มออกจากระบบที่ Sidebar ล้าง session แล้ว redirect กลับหน้า Login

---

## Requirement 2: Dashboard (แดชบอร์ด)
### User Stories
- ในฐานะผู้ใช้งาน ฉันต้องการเห็นภาพรวมของการแจ้งซ่อมทั้งหมด เพื่อติดตามสถานะ

### Acceptance Criteria
1. แสดง Stat Cards 4 ใบ: ทั้งหมด, รอดำเนินการ, กำลังดำเนินการ, เสร็จสิ้น
2. แสดงตารางรายการแจ้งซ่อมล่าสุด 5 รายการ เรียงตามวันที่ล่าสุด
3. ตารางแสดงข้อมูล: รหัส, รายการ, สถานที่, สถานะ (badge สี), วันที่
4. ข้อมูลอัปเดตทุกครั้งที่เข้าหน้า Dashboard

---

## Requirement 3: Maintenance Request (แจ้งซ่อม)
### User Stories
- ในฐานะผู้ใช้งาน ฉันต้องการแจ้งซ่อมใหม่ได้ เพื่อแจ้งปัญหาที่พบ
- ในฐานะผู้ใช้งาน ฉันต้องการแก้ไขและลบรายการแจ้งซ่อมได้

### Acceptance Criteria
1. แสดงตารางรายการแจ้งซ่อมทั้งหมด พร้อมคอลัมน์: รหัส, รายการ, สถานที่, ประเภท, สถานะ, วันที่, จัดการ
2. มีปุ่ม "แจ้งซ่อมใหม่" เปิด Modal ฟอร์ม
3. ฟอร์มประกอบด้วย: รายการซ่อม, สถานที่, ประเภท (ไฟฟ้า/ประปา/แอร์/อาคาร/อื่นๆ), รายละเอียด
4. บันทึกรายการใหม่ สร้างรหัสอัตโนมัติ (REQ-XXX) สถานะเริ่มต้น "รอดำเนินการ" วันที่ปัจจุบัน
5. กดปุ่ม "แก้ไข" เปิด Modal พร้อมข้อมูลเดิม สามารถแก้ไขและบันทึกได้
6. กดปุ่ม "ลบ" แสดง confirm dialog ก่อนลบ
7. ข้อมูลเก็บใน localStorage

---

## Requirement 4: Report (รายงาน)
### User Stories
- ในฐานะผู้ใช้งาน ฉันต้องการดูรายงานสรุปการแจ้งซ่อม เพื่อวิเคราะห์ข้อมูล

### Acceptance Criteria
1. แสดง Stat Cards: จำนวนทั้งหมด, อัตราเสร็จสิ้น (%), ประเภทที่แจ้งมากสุด
2. แสดงตารางสรุปตามประเภท: ประเภท, จำนวน, เสร็จสิ้น, รอดำเนินการ
3. ข้อมูลคำนวณจากรายการแจ้งซ่อมทั้งหมดใน localStorage

---

## Requirement 5: Settings (ตั้งค่า)
### User Stories
- ในฐานะผู้ใช้งาน ฉันต้องการตั้งค่าระบบได้ เช่น การแจ้งเตือนและภาษา

### Acceptance Criteria
1. แสดงหมวดการแจ้งเตือน: toggle แจ้งเตือนอีเมล, toggle แจ้งเตือนในระบบ
2. แสดงหมวดทั่วไป: เลือกภาษา (ไทย/English), เลือกจำนวนรายการต่อหน้า (10/25/50)
3. Toggle สามารถเปิด/ปิดได้

---

## Requirement 6: Layout & Navigation (โครงสร้างและเมนู)
### User Stories
- ในฐานะผู้ใช้งาน ฉันต้องการเมนูนำทางที่ใช้งานง่าย เพื่อเข้าถึงทุกฟีเจอร์

### Acceptance Criteria
1. Sidebar แสดงเมนู 4 รายการ: แดชบอร์ด, แจ้งซ่อม, รายงาน, ตั้งค่า
2. เมนูที่ active แสดง highlight
3. คลิกเมนูเปลี่ยนหน้าแบบ SPA (ไม่ reload)
4. แสดงข้อมูลผู้ใช้ที่ด้านล่าง Sidebar (ชื่อ, role, avatar)
5. Responsive: บนมือถือมีปุ่ม hamburger เปิด/ปิด Sidebar
